# Fedorov_DevOps24_1m_Marketplaces

Расширение для конфигурации "1С:Управление торговлей", автоматизирующее процессы работы с маркетплейсами **Ozon**, **Wildberries** и **Яндекс.Маркет**.

## Возможности

- Работа с Ozon
- Работа с Wildberries
- Работа с Яндекс Маркет
- Работа с несколькими аккаунтами
- Сопоставление складов
- Сопоставление товаров
- Создание номенклатуры в 1С на основании товаров на маркетплейсе
- Загрузка остатков из 1С в МП
- Ввод остатков в 1С из МП
- Управление ценами
- Загрузка заказов
- Обработка заказов FBS
- Групповая обработка заказов FBS
- Печать этикеток
- Передача маркировок на маркетплейс
- Создание поставок FBO
- Загрузка отчётов комиссионера
- Интеграция напрямую с маркетплейсами

## Требования

| Компонент | Минимальная версия |
|---|---|
| 1С:Предприятие | 8.3.25 и выше |
| 1С:Управление торговлей | 11.5.19.63 и выше |
| 1C:Enterprise Development Tools (EDT) | 2025.1.5 и выше |

Для работы расширения необходима действующая клиентская лицензия 1С — она уже используется основной конфигурацией, отдельная лицензия на расширение не требуется.

## Структура проекта

```
UT_Marketplaces.Marketplaces/   — исходный код расширения в формате EDT
.github/
  workflows/
    deploy-test.yml              — автоматическое развёртывание на тестовую базу
    deploy-prod.yml              — ручное развёртывание на продуктивную базу
```

## Ветвление

| Ветка | Назначение |
|---|---|
| `develop` | Разработка и тестирование. Все изменения попадают сюда. |
| `main` | Итоговая стабильная версия расширения. Обновляется из `develop` после проверки. |

## CI/CD

### Тестовая база

Автоматическое развёртывание выполняется два раза в сутки:

| Расписание (UTC) | По Москве | Когда |
|---|---|---|
| `0 22` | 01:00 | Ночное обновление |
| `0 10` | 13:00 | Дневное обновление |

Воркфлоу автоматически:
1. Проверяет наличие новых коммитов в `develop` за последние 12 часов (при установленном параметре "Обновлять даже без изменений")
2. Конвертирует проект из формата EDT в XML
3. Устанавливает блокировку базы и ожидает завершения сеансов
4. Создаёт резервную копию текущего расширения (.cfe)
5. Загружает новое расширение и применяет изменения
6. При ошибках — автоматически откатывается на предыдущую версию
8. Снимает блокировку базы

Также доступен ручной запуск с параметрами:
- **Обновить даже без изменений** — принудительный деплой
- **Через сколько минут заблокировать базу** — задержка перед блокировкой (по умолчанию 30 мин)
- **Длительность окна обновления** — время блокировки (по умолчанию 30 мин)

### Продуктивная база

Воркфлоу автоматически:
1. Расчитывает время блокировки базы, устанавливает блокировку базы и ожидает завершения сеансов
2. Конвертирует проект из формата EDT в XML
4. Создаёт резервную копию текущего расширения (.cfe)
5. Загружает новое расширение и применяет изменения
6. При ошибках — автоматически откатывается на предыдущую версию
7. Применяет изменения
8. Снимает блокировку базы

Развёртывание выполняется **только вручную** с параметрами:
- **Время начала обновления (HH:MM, например 23:00)** — время устаноки блокировки входа в базу
- **За сколько минут до обновления закрыть вход в базу** — задержка перед блокировкой (по умолчанию 30 мин)
- **Длительность окна обновления (минут)** — время блокировки (по умолчанию 60 мин)
